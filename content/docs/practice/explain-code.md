## 코드를 몰라도 AI가 쉽게 설명해줍니다

"이 코드가 뭐 하는 거예요?"라고 물으면 클로드코드가 쉽게 설명해줍니다. 코딩 경험이 없어도 괜찮습니다.

---

## 실전 예시: 코드 이해하기

### 내 요청

```
app.py 파일을 열어서 이게 뭐 하는 코드인지 쉽게 설명해줘.
초등학생도 이해할 수 있게!
```

### Claude의 응답

```
app.py를 읽어봤어요! 🎉

이 코드는 "숫자 계산기"예요.

• 사용자가 숫자 2개를 입력하면
• 더하기, 빼기, 곱하기를 해서
• 결과를 화면에 보여줘요!

마치 계산기 앱처럼 동작하는 거예요 😊
```

**결과:** 복잡한 코드를 쉽게 이해했습니다! ✨

---

## 이렇게 요청해보세요

### 📖 이해를 위한 요청

```
"이 코드가 무슨 일을 하는지 설명해줘"
→ 코드의 전체 목적 설명

"이 함수가 왜 필요한지 알려줘"
→ 함수의 역할 설명

"비개발자도 이해할 수 있게 설명해줘"
→ 쉬운 언어로 설명
```

### 🔍 구조 파악을 위한 요청

```
"이 프로젝트 폴더 구조 설명해줘"
→ 어떤 파일들이 있는지 정리

"어떤 파일이 어떤 역할을 하는지"
→ 각 파일의 목적 설명

"핵심 파일이 뭔지 알려줘"
→ 가장 중요한 파일 파악
```

## 다양한 상황에서 활용하기

### 1️⃣ GitHub에서 받은 코드 분석

```
"GitHub에서 받은 이 코드가 뭐 하는 건지 설명해줘"
→ 오픈소스 코드 이해

"이 코드를 어떻게 사용하는지 알려줘"
→ 사용 방법 설명
```

### 2️⃣ 이전에 만든 내 코드 분석

```
"내가 3개월 전에 만든 이 파일이 뭐 했는지 기억이 안 나"
→ 과거 코드 이해

"이 코드가 완성된 건가, 뭐가 빠진 건가?"
→ 코드 상태 파악
```

### 3️⃣ 한 줄씩 상세 설명

```
"각 줄이 뭐 하는지 자세히 설명해줄 수 있을까?"
→ 라인별 분석

"이 문법이 뭐예요?"
→ 특정 개념 설명
```

## 설명받은 후 개선하기

### 흐름 예시

```
1단계: 코드 설명 받기
> 이 코드가 뭐 하는 거야?
✓ (Claude가 설명)

2단계: 이해한 후 개선 요청
> 그럼 이 부분을 더 간단하게 만들 수 있을까?
✓ (Claude가 수정)

3단계: 추가 기능 요청
> 이제 데이터를 저장할 수 있게 해줄래?
✓ (Claude가 코드 추가)
```

## 📚 설명의 수준 조절하기

### 초보자 수준

```
"컴퓨터 몰라도 이해할 수 있게 설명해줘"
→ 매우 간단한 설명

"10살 아이도 알 수 있게"
→ 어려운 용어 피함
```

### 중간 수준

```
"개발자 입문자도 이해할 수 있게"
→ 기술 용어 포함하지만 설명 추가

"왜 이렇게 작성했는지 이유 포함해줘"
→ 논리적 설명
```

### 심화 수준

```
"성능 최적화 관점에서 분석해줘"
→ 효율성 분석

"이 코드의 문제점을 찾아줘"
→ 버그나 개선점 지적
```

## 💡 팁: 설명을 요청할 때

### 1. 정확한 파일명 제시

```
❌ "이 코드 설명해줘"
✅ "main.py 파일 설명해줘"
```

### 2. 설명 수준 명시

```
❌ "뭐 하는 거야?"
✅ "비개발자가 이해할 수 있게 설명해줄래?"
```

### 3. 특정 부분만 설명받기

```
✅ "이 함수만 설명해줄래?"
✅ "34번 줄부터 40번 줄까지 설명해줄래?"
```
