## 슬래시(/) 명령어란?

일반적인 대화 대신 `/` 기호로 시작하는 특수 명령어입니다. 더 빠르고 간결하게 작업을 제어할 수 있습니다.

---

## 📚 기본 명령어 10가지

### 1️⃣ `/help` — 도움말 보기

```bash
/help
```

**역할:** 사용 가능한 모든 명령어 목록 확인
- 처음 사용할 때 명령어 기억이 안 나면 입력
- Claude Code가 모든 옵션을 알려줍니다

---

### 2️⃣ `/clear` — 대화 초기화

```bash
/clear
```

**역할:** 이전 대화 내용을 지우고 새로 시작
- 다른 주제로 넘어갈 때 입력하면 좋습니다
- 이전 대화 맥락을 완전히 제거합니다
- 비용 절감 효과도 있습니다

**사용 예시:**
```
> 파이썬 코드 만들어줘
> (여러 번 대화)
> /clear
> 이제 HTML 웹사이트 만들어줄래?
```

---

### 3️⃣ `/status` — 현재 상태 확인

```bash
/status
```

**역할:** 사용 중인 모델과 비용 현황 표시
- 지금까지 얼마나 사용했는지 확인
- 남은 크레딧 확인 (유료 사용자)
- API 사용량 모니터링

---

### 4️⃣ `/compact` — 대화 압축

```bash
/compact
```

**역할:** 대화가 길어질 때 요약해서 비용 절약
- 오래된 대화를 요약으로 변환
- 토큰 사용량 감소
- 대화 맥락 유지하면서 비용 절감

---

### 5️⃣ `/init` — 프로젝트 CLAUDE.md 자동 생성

```bash
/init
```

**역할:** 프로젝트 설정 파일을 자동으로 생성
- 클로드코드 사용자 지정 설정
- 프로젝트별 규칙 저장
- 다음번 실행 시 자동 적용

**언제 사용?**
- 새 프로젝트 시작할 때
- 팀원들과 같은 규칙으로 작업하고 싶을 때
- "이 프로젝트는 항상 이렇게 해줘"라는 규칙 설정

**사용 예시:**
```
> /init
Claude가 CLAUDE.md 파일 생성
그 안에 프로젝트 규칙을 자동으로 저장
다음부터 자동으로 적용됨!
```

---

### 6️⃣ `/model` — AI 모델 변경

```bash
/model
```

**역할:** Claude Code가 사용할 AI 모델 선택
- Claude 3.5 Sonnet (기본, 빠름)
- Claude 3 Opus (더 강력함, 복잡한 작업)
- 더 저렴한 모델 선택 가능

**언제 사용?**
- 복잡한 작업이 필요할 때
- 비용을 절감하고 싶을 때
- 특정 모델이 더 잘 작동할 때

**사용 예시:**
```
> /model
Claude가 선택 가능한 모델들 보여줌:
1. Claude 3.5 Sonnet (현재)
2. Claude 3 Opus
3. Claude 3 Haiku (더 저렴)

> 2 선택
이제 Opus 모델 사용!
```

---

### 7️⃣ `/compact` — 대화 내용 압축

```bash
/compact
```

**역할:** 긴 대화를 요약해서 비용 절약
- 오래된 대화를 정리
- 토큰 사용량 감소 (비용 절감)
- 대화 맥락은 유지하면서 압축

**언제 사용?**
- 같은 주제로 너무 많이 대화했을 때
- 비용이 크게 나올 것 같을 때
- 대화 기록을 정리하고 싶을 때

**비용 절감 효과:**
- 보통 40~60% 토큰 감소
- 예: $10 드는 대화 → 압축 후 $4~6

---

### 8️⃣ `/diff` — 변경 파일 확인하기

```bash
/diff
```

**역할:** 마지막으로 수정한 파일들의 차이점 표시
- 뭐가 바뀌었는지 정확히 보기
- 실수로 삭제된 부분 확인
- "이 수정이 맞나?" 검증

**사용 예시:**
```
> /diff
파일들의 변경 내역을 보여줌:
- 삭제된 줄 (빨강)
+ 추가된 줄 (초록)
! 수정된 줄
```

---

### 9️⃣ `/rewind` — 이전 상태로 되돌리기

```bash
/rewind
```

**역할:** 마지막 변경사항을 취소하고 이전 버전으로 돌아가기
- "이 수정 실수했다" → 복구
- 마지막 상태로 돌아가기
- 실패한 실험 되돌리기

**언제 사용?**
- AI가 잘못 수정했을 때
- "다시 원래대로 해줄래?" 대신 사용
- 빠른 복구

---

### 🔟 `Ctrl+C` — 작업 중지

터미널에서 `Ctrl` 키와 `C` 키를 **동시에** 누르기

**역할:** AI가 실행 중인 작업을 즉시 멈춤
- 잘못된 방향으로 가고 있으면 중단
- 파일 생성 중간에 멈추기
- 오류가 발생했을 때 중단

**사용 예시:**
```
> 1000개 파일 만들어줘
(AI가 실행 중...)
Ctrl+C 누르기
작업이 멈춥니다
```

## 🚪 종료하고 다시 시작하기

### 종료 방법

두 가지 방법이 있습니다:

```bash
/exit
```

또는

```bash
Ctrl+D
```

### 다시 시작하려면

터미널에서 다시 입력:

```bash
claude
```

자동으로 이전 로그인이 유지되므로 추가 로그인 필요 없습니다.

## 💡 명령어 사용 팁

### 조합 사용하기

```
> 웹사이트 만들어줘
(여러 번 수정 요청)
> /status   (비용 확인)
> /compact  (대화 압축)
> /clear    (새로운 주제 시작)
```

### 언제 사용하면 좋은가?

| 상황 | 명령어 |
|------|--------|
| 명령어 기억 안 남 | `/help` |
| 새로운 주제 시작 | `/clear` |
| 비용 확인하고 싶음 | `/status` |
| 대화가 너무 김 | `/compact` |
| 프로젝트 규칙 설정 | `/init` |
| 더 강한 모델 필요 | `/model` |
| 뭐가 바뀌었는지 보기 | `/diff` |
| 마지막 수정 취소 | `/rewind` |
| AI 작업이 잘못됨 | `Ctrl+C` |
