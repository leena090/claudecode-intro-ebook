## 에이전트 팀이란?

**에이전트 팀**은 여러 AI가 함께 작업하는 시스템입니다. 마치 회사에서 각자 다른 역할을 하는 팀원들이 협력하듯이, 각 AI가 전문 역할을 맡아 복잡한 작업을 빠르게 해결합니다.

---

## 팀 구성 원리

### 한 명으로 처리 (느림)
```
당신 → AI(모든 작업) → 결과
     ↑   ↑   ↑
   느림  비효율  시간 오래 걸림
```

### 팀으로 처리 (빠름)
```
당신 → 팀 리더(AI)
     ├→ 연구원(검색)
     ├→ 작가(글쓰기)
     ├→ 검수원(검토)
     └→ 결과 (병렬 처리 = 빠름!)
```

<div class="note-circle">
  ○ 팀 작업은 동시에 진행되어 시간이 크게 단축됩니다
</div>

---

## 팀의 역할 분담

### 일반적인 팀 구성

| 역할 | 역할 설명 | 담당 작업 |
|------|----------|---------|
| **팀 리더** | 전체 조율 | 계획 수립, 결과 통합 |
| **연구원** | 정보 수집 | 검색, 조사, 분석 |
| **개발자** | 코딩 | 구현, 테스트, 디버깅 |
| **작가** | 문서 작성 | 튜토리얼, 가이드 |
| **검수원** | 품질 보증 | 검토, 검증, 피드백 |

---

## 팀 구성 예시

### 웹사이트 개발 팀

```
팀 리더 (당신)
├─ 디자이너 에이전트
│   └ UI/UX 설계, 레이아웃
├─ 프론트엔드 에이전트
│   └ HTML/CSS/JavaScript 개발
├─ 백엔드 에이전트
│   └ API, 데이터베이스 구현
└─ QA 에이전트
    └ 테스트, 버그 검증
```

---

### 문서 작성 팀

```
팀 리더 (당신)
├─ 연구 에이전트
│   └ 주제 조사, 정보 수집
├─ 작가 에이전트
│   └ 콘텐츠 작성
├─ 편집 에이전트
│   └ 문법, 스타일 수정
└─ SEO 에이전트
    └ 검색어 최적화
```

## 팀 구성 방법

### 단계 1: 팀 리더로 작업

```bash
claude --team
```

### 단계 2: 팀원 초대

```
> @recruiter "이 프로젝트에 어울릴 3명의 에이전트를 찾아줄래?"

팀 리더가 필요한 전문가들을 찾습니다.
```

### 단계 3: 작업 할당

```
> @developer "홈페이지 HTML 구조 만들어줄래?"
> @designer "색상 테마 정해줄래?"
> @qa "테스트 케이스 작성해줄래?"
```

### 단계 4: 결과 통합

팀원들이 각자 작업하고, 팀 리더가 결과를 모아 최종 산출물을 만듭니다.

---

## 팀 사용의 장점

### 1. ⚡ 속도

```
혼자: 1시간 30분 (순차 작업)
팀: 30분 (병렬 작업)
└─ 3배 빠름!
```

---

### 2. 🎯 전문성

각 팀원이 특화된 역할에만 집중:
- 개발자는 코딩만
- 작가는 글쓰기만
- 검수원은 품질 관리만

**결과:** 더 높은 품질

---

### 3. 🔄 견제 및 개선

```
개발자: "코드를 이렇게 짰습니다"
검수원: "여기 버그가 있는데?"
개발자: "아, 수정하겠습니다"

└─ 상호 검증으로 더 완벽한 결과
```

## 팀 협업 예시

### 예시 1: 블로그 포스트 작성

```
당신: "블로그 포스트를 작성해줄 팀을 구성해줄래?"

팀 리더가 팀 구성:
- 연구원 (@researcher): 주제 조사
- 작가 (@writer): 글 작성
- 편집자 (@editor): 검토 및 수정

당신: "@researcher 'AI의 미래' 주제로 조사해줄래?"
연구원: (웹 검색) → 최신 자료 5개 제시

당신: "@writer 이 자료를 바탕으로 포스트 작성해줄래?"
작가: (글 작성 완료)

당신: "@editor 문법 확인해줄래?"
편집자: (3개 오류 수정, 완성)
```

---

### 예시 2: 소프트웨어 개발

```
당신: "쇼핑몰 앱을 만드는 팀을 꾸려줄래?"

팀 구성:
- 개발자 (@frontend): UI 개발
- 개발자 (@backend): API 개발
- QA (@tester): 테스트

병렬 진행:
@frontend: (로그인 화면 개발) → 30분
@backend:  (사용자 API 개발)  → 30분
@tester:   (테스트 시나리오 작성) → 30분

결과: 3가지 작업이 동시에 진행되어 전체 시간 단축
```

## 팀 내 커뮤니케이션

### 팀 채널에서 메시지

```
당신: "@developer 이 부분은 어떻게 생각해?"

개발자: "이렇게 하면 성능이 더 좋을 것 같습니다."

당신: "@qa 개발자 제안을 검증해줄 수 있나?"

QA: "네, 테스트한 결과 20% 성능 향상이 확인됩니다."
```

## 💡 쉽게 이해하기

팀 시스템은 **영화 촬영 팀**과 같습니다.

**혼자서:**
- 배우 역할 + 감독 역할 + 촬영 역할 = 비효율

**팀으로:**
- 배우: 연기만 집중
- 감독: 연출만 관리
- 촬영팀: 촬영만 담당
└─ 각자 최고의 결과를 만듦 = 완성도 높은 영화

마찬가지로:
- 개발자는 코딩만
- 디자이너는 디자인만
- 테스터는 테스트만
└─ 고품질의 완성된 제품

## 팀 작업 모범 사례

### ✅ 좋은 팀 구성
```
- 명확한 역할 분담
- 적절한 팀 크기 (3~5명)
- 정기적인 커뮤니케이션
- 공통 목표 설정
```

### ❌ 피할 것
```
- 역할 겹치는 팀원 많기
- 너무 큰 팀 (10명 이상)
- 팀원 간 소통 부족
- 목표 불명확
```

```bash

```

