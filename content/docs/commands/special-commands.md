## 특수 명령어란?

기본적인 작업을 넘어서 더 고급스럽고 특별한 기능들을 제어하는 명령어입니다. 프로젝트 관리, 코드 검토, 버전 관리 등을 할 때 사용합니다.

---

## 특수 명령어 9가지

### 1️⃣ `/init` — 새 프로젝트 초기화

```bash
/init "프로젝트명"
```

**역할:** 새로운 프로젝트를 위한 초기 설정 및 폴더 구조 생성

<div class="note-circle">
○ 새 프로젝트 시작할 때 가장 먼저 해야 할 명령어입니다
</div>

**사용 예시:**
```bash
/init "쇼핑몰 웹사이트"
```

**자동으로 생성되는 항목:**
- 프로젝트 폴더 구조
- 설정 파일들
- 시작 스켈레톤 코드
- 프로젝트 문서

**이후 작업:**
```bash
> 이제 쇼핑몰 홈페이지 HTML 만들어줄래?
(AI가 프로젝트 구조 인식)
```

---

### 2️⃣ `/plan` — 프로젝트 계획 수립

```bash
/plan
```

**역할:** 전체 프로젝트의 단계별 계획을 작성

**사용 예시:**
```bash
> 온라인 쇼핑몰 만들 거야
> /plan
```

**자동으로 생성되는 항목:**
- 전체 개요 (무엇을 만들 건가)
- 필요한 기술 스택
- 단계별 일정
- 각 단계별 작업 내용
- 예상 기간

**계획 예시:**
```
1단계: 기획 (1주)
- UI/UX 디자인
- 기술 스택 결정

2단계: 프론트엔드 (2주)
- HTML/CSS 작성
- React 컴포넌트

3단계: 백엔드 (2주)
- API 설계
- 데이터베이스 구성

4단계: 통합 & 테스트 (1주)
- 전체 통합
- 버그 수정

5단계: 배포 (1주)
- 서버 배포
- 모니터링
```

---

### 3️⃣ `/fast` — 빠른 모드 활성화

```bash
/fast
```

**역할:** 응답 속도를 최우선으로 설정

<mark>더 빠르지만 덜 정확할 수 있습니다</mark>

**사용 예시:**
```bash
> 간단한 질문만 하고 싶어
> /fast
(응답이 훨씬 빨라짐)
```

**활용:**
- 빠른 피드백이 필요한 경우
- 단순한 질문만 할 때
- 비용을 절감하고 싶을 때

**주의:**
```bash
> /fast
> 다시 정상 모드로
> /fast off
```

---

### 4️⃣ `/voice` — 음성 모드 시작

```bash
/voice
```

**역할:** 음성으로 Claude Code에 명령하기

**사용 예시:**
```bash
> /voice
(마이크가 활성화됨)
> "파이썬 함수 만들어줄래?" (음성 입력)
(AI가 음성 인식하고 응답)
```

**활용:**
- 손이 바쁠 때 (요리 중, 운전 중)
- 타이핑하기 싫을 때
- 자연스러운 대화 원할 때

**끝내기:**
```bash
> /voice off
(또는 음성 입력 없이 엔터)
```

### 5️⃣ `/effort` — 작업량 조절

```bash
/effort level
```

**역할:** AI의 작업 상세도를 조절

**레벨:**
- `low` — 최소한만 (빠르고 저렴)
- `medium` — 적당한 수준 (기본)
- `high` — 매우 상세 (느리지만 정확)

**사용 예시:**
```bash
/effort low
> 간단한 요약만 해줄래?

/effort high
> 이건 정말 중요한 부분이니까 상세하게 설명해줄래?

/effort medium
> 보통 수준으로 돌아가
```

**토큰 사용량:**
- `low` — 30% 절감
- `medium` — 기본 (100%)
- `high` — 50% 더 사용

---

### 6️⃣ `/sandbox` — 안전한 격리 환경에서 실행

```bash
/sandbox
```

**역할:** 위험한 코드를 격리된 환경에서 테스트

<div class="note-star">
★ 컴퓨터 손상을 방지하는 보안 기능입니다
</div>

**사용 예시:**
```bash
> 이 코드가 안전한지 확인해줄래?
> /sandbox
(격리 환경에서 실행, 컴퓨터 보호)
```

**보호 대상:**
- 파일 시스템 (파일 삭제 방지)
- 시스템 권한 (시스템 수정 방지)
- 네트워크 (악의적 통신 방지)

**Sandbox 종료:**
```bash
/sandbox off
```

### 7️⃣ `/pr` — PR(Pull Request) 자동 생성

```bash
/pr "제목" "설명"
```

**역할:** GitHub 등의 저장소에 PR(병합 요청)을 자동 생성

<mark>GitHub에 연결되어 있어야 사용 가능합니다</mark>

**사용 예시:**
```bash
/pr "홈페이지 레이아웃 개선" "모바일 반응형 추가"
```

**자동으로 생성되는 항목:**
- PR 제목
- 상세 설명
- 변경 파일 목록
- 변경사항 요약
- 체크리스트

**활용:**
- 팀원에게 코드 검토 요청
- 변경사항을 깔끔하게 정리
- Git 작업 자동화

---

### 8️⃣ `/review` — 코드 리뷰 받기

```bash
/review
```

**역할:** AI가 작성한 코드를 전문가처럼 검토

**검토 항목:**
- 코드 품질
- 버그 가능성
- 성능 최적화
- 보안 문제
- 가독성 개선안

**사용 예시:**
```bash
> 파이썬 함수 만들어줘
(AI가 함수 작성)
> /review
```

**리뷰 결과:**
```
✓ 기능: 정상 작동
⚠ 성능: for 루프를 list comprehension으로 개선 가능
✗ 보안: 입력값 검증 필요
💡 가독성: 변수명이 명확하지 않음

개선안:
...
```

---

### 9️⃣ `/commit` — Git 커밋 메시지 생성

```bash
/commit
```

**역할:** 변경사항을 분석해서 의미 있는 커밋 메시지 자동 생성

<mark>Git 저장소가 있어야 사용 가능합니다</mark>

**사용 예시:**
```bash
> (여러 파일 수정)
> /commit
```

**자동 생성되는 메시지:**
```
feat: 사용자 인증 기능 추가

- JWT 토큰 기반 인증 구현
- 로그인/로그아웃 API 추가
- 비밀번호 해싱 처리
- 보안 토큰 관리

Closes #123
```

**구조:**
- 타입: feat (기능), fix (버그), docs (문서) 등
- 제목: 한 줄 요약
- 본문: 상세 변경사항
- 이슈 연결

## 🚀 특수 명령어 활용 흐름

### 새 프로젝트 시작하기

```bash
Step 1: 프로젝트 초기화
/init "쇼핑몰 프로젝트"

Step 2: 계획 수립
/plan

Step 3: 개발 시작
> 홈페이지 HTML 만들어줘

Step 4: 코드 검토
/review

Step 5: 리뷰 반영
> 리뷰 결과 개선해줄래?

Step 6: PR 생성
/pr "홈페이지 완성" "모든 요구사항 충족"

Step 7: 커밋
/commit
```

## 💡 쉽게 이해하기

### 특수 명령어를 건설 프로젝트로 비유하면?

Claude Code의 특수 명령어는 **건물을 짓는 전체 과정**과 같습니다:

- **`/init`** = 건설 부지 정리 및 기초 설계
- **`/plan`** = 건설 계획 수립 (언제까지, 어떻게)
- **`/fast`** = 빠른 시공 (품질 조금 포기)
- **`/voice`** = 현장 감독 음성 명령
- **`/effort`** = 시공 정도 (기본/상급/초급)
- **`/sandbox`** = 안전 테스트 현장
- **`/pr`** = 시공 완료 보고서 (건축주 승인)
- **`/review`** = 전문가 검수
- **`/commit`** = 건설 공정 기록

### 예를 들어...

**상황 1: 큰 프로젝트 시작할 때**
```bash
> 온라인 쇼핑몰을 처음부터 만들어야 해
/init "쇼핑몰"
/plan
→ 전체 계획이 자동으로 세워짐
→ 이제 단계별로 따라가면 됨
```

**상황 2: 코드가 정말 중요할 때**
```bash
> 결제 시스템 코드 만들어줄래? (돈이 오가는 곳)
/effort high
→ 매우 상세하고 안전한 코드 생성

/review
→ 전문가 수준의 검토

/sandbox
→ 안전한 환경에서 테스트
```
