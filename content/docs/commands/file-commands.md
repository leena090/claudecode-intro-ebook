## 파일 & 코드 명령어란?

Claude Code로 파일을 만들고, 수정하고, 관리할 때 사용하는 명령어들입니다. 효율성을 높이고 실수를 줄이는 데 도움이 됩니다.

---

## 파일 & 코드 명령어 5가지

### 1️⃣ `/diff` — 파일 변경사항 비교하기

```bash
/diff
```

**역할:** 이전 파일과 현재 파일의 차이점을 시각적으로 표시

<div class="note-circle">
○ 코드 수정 전후를 명확하게 비교할 수 있습니다
</div>

**사용 예시:**
```bash
> 파일을 수정해줘
(AI가 파일 수정)
> /diff
(무엇이 바뀌었는지 보여줌)
```

**표시되는 내용:**
- 빨간색 `-` : 삭제된 부분
- 초록색 `+` : 추가된 부분
- 회색 부분 : 변경되지 않은 부분

**사용 이유:**
- 실수 방지 (수정 전에 확인)
- 변경 내용 명확히 파악
- 필요 시 수정 요청 가능

---

### 2️⃣ `/copy` — 코드 복사하기

```bash
/copy "파일명.확장자"
```

**역할:** Claude Code가 생성한 코드를 클립보드에 복사

<mark>다른 에디터(VS Code 등)에 붙여넣을 때 매우 유용합니다</mark>

**사용 예시:**
```bash
> HTML 코드 만들어줘
(AI가 index.html 생성)
> /copy "index.html"
(클립보드에 복사됨)
> Ctrl+V로 붙여넣기
```

**편의성:**
- 한 번에 전체 코드 복사
- 복사 실수 방지
- 여러 파일 동시 복사 가능

---

### 3️⃣ `/export` — 파일 내보내기 (다운로드)

```bash
/export "파일명.확장자"
```

**역할:** Claude Code에서 생성한 파일을 내 컴퓨터에 다운로드

**사용 예시:**
```bash
> 웹사이트 코드 만들어줘
(AI가 여러 파일 생성)
> /export "index.html"
> /export "style.css"
> /export "script.js"
(각 파일이 다운로드됨)
```

**활용 방법:**
1. Claude Code에서 파일 생성
2. `/export`로 다운로드
3. 내 컴퓨터에서 사용
4. 파일 수정 후 다시 업로드해서 도움받기

**여러 파일 한 번에 내보내기:**
```bash
/export all
(모든 파일을 한 번에 다운로드)
```

---

### 4️⃣ `/compact` — 대화 압축하기

```bash
/compact
```

**역할:** 길어진 대화를 요약으로 변환해서 토큰 절약

<div class="note-star">
★ 비용을 크게 절감할 수 있는 매우 유용한 명령어입니다
</div>

**언제 사용할까?**
- 대화가 50번 이상 반복되었을 때
- 비용이 걱정될 때
- 응답이 느려졌을 때 (메모리 압박)

**사용 예시:**
```bash
> 파이썬 배우기 (대화 30번)
> /compact
(오래된 내용을 요약으로 변환)
> 자, 이제 다음 주제로 진행해줄래?
(새로운 대화 시작, 토큰 사용량 ↓)
```

**압축 효과:**
- 토큰 사용량 30~50% 감소
- 응답 속도 향상
- AI가 필수 내용만 기억

**주의:**
- 압축 후에도 대화 맥락은 유지됩니다
- 오래된 세부사항은 일부 손실될 수 있습니다

---

### 5️⃣ `/memory` — 기억 관리하기

```bash
/memory
```

**역할:** AI가 기억해야 할 중요한 정보를 등록하고 관리

**활용 사례:**
```bash
/memory add "내 이름: 김철수"
/memory add "직업: 프로그래머"
/memory add "관심사: Python, 웹개발"
```

그러면 다음 대화에서:
```bash
> 안녕하세요
(AI 응답) "안녕하세요, 철수님! Python 웹개발 관련해서 도와드릴게요."
```

**메모리 확인:**
```bash
/memory list
(등록된 모든 정보 확인)
```

**메모리 삭제:**
```bash
/memory delete "내 이름: 김철수"
```

**활용 팁:**
- 개인 정보 (이름, 직업, 관심사)
- 프로젝트 특수 정보
- 자주 반복되는 설정

## 🛠️ 파일 작업 실전 예시

### 예시 1: 웹사이트 만들기

```bash
Step 1: 요청
> HTML 웹사이트 만들어줘

Step 2: 확인
> /diff
(변경사항 확인)

Step 3: 복사
> /copy "index.html"
> /copy "style.css"

Step 4: 내 컴퓨터에서 사용
(VS Code 등에 붙여넣기)

Step 5: 추가 수정
> 이 부분을 다르게 해줄래?

Step 6: 내보내기
> /export "index.html"
(수정된 파일 다운로드)
```

### 예시 2: 대화 길어질 때

```bash
Step 1: Python 배우기 (대화 30번)

Step 2: 비용 확인
> /status
(토큰 많이 사용됨)

Step 3: 압축하기
> /compact

Step 4: 새로운 주제
> 자 이제 JavaScript 배워봅시다
(더 저렴하게 계속 진행)
```

## 💡 쉽게 이해하기

### 파일 명령어를 문서 작성으로 비유하면?

Claude Code의 파일 명령어는 **문서 작성 프로그램(MS Word 같은 것)**과 유사합니다:

- **`/diff`** = 변경사항 추적 (Track Changes) 기능
- **`/copy`** = Ctrl+C (복사)
- **`/export`** = 다른 형식으로 저장 (PDF, Word 등)
- **`/compact`** = 문서 압축 (용량 줄이기)
- **`/memory`** = 저장된 서식/템플릿 기억

### 예를 들어...

**상황 1: 코드를 확인하고 싶을 때**
```bash
> HTML 코드 만들어줘
(AI가 100줄의 코드 생성)
> 뭐가 바뀌었는지 정확히 보고 싶은데...
/diff
→ 정확한 변경사항이 화면에 표시됨
```

**상황 2: 코드를 내 에디터에 붙여넣고 싶을 때**
```bash
> 파이썬 함수 만들어줘
(AI가 코드 생성)
> 이걸 내 VS Code에 붙여넣고 싶어
/copy "function.py"
→ 클립보드에 복사, Ctrl+V로 붙여넣기
```

**상황 3: 마지막으로 작업한 파일을 다운로드하고 싶을 때**
```bash
> /export "website.html"
→ 내 컴퓨터의 Downloads 폴더에 저장됨
```

## 파일 명령어 활용 팁

### Tip 1: 수정 전에 항상 /diff로 확인
```bash
> 이 부분 고쳐줘
(수정 결과)
> /diff
(먼저 확인 후 수락)
```

### Tip 2: 대화가 길어지면 자주 /compact
```bash
> (대화 20번 후)
> /compact
(비용 절감 + 응답 속도 ↑)
```

### Tip 3: 중요한 정보는 /memory에 저장
```bash
/memory add "프로젝트명: 쇼핑몰"
/memory add "기술스택: React + Node.js"
(다음 대화에서 자동 인지)
```
